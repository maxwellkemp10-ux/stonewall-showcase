# Stonewall Showcase

> Stonewall is a knowledge-control system for complex, high-consequence work. **Platform v10.**

> ℹ️ Built for teams that cannot afford drift, delay, or fragmented operating memory.

### Executive premise

Most organizations do not suffer from a lack of information.

They suffer from weak control over information.

Critical knowledge becomes fragmented.

Ownership becomes diffuse.

Search replaces structure.

Rework replaces continuity.

> ✅ Stonewall is designed for live use, not archive storage.

### Strategic thesis

Stonewall converts knowledge from a passive repository into an active operating asset.

It introduces structure where most systems tolerate sprawl.

That shift improves speed, consistency, and execution quality.

### What it does

* Consolidates fragmented inputs into one governed system.
* Preserves chronology, naming, and source integrity.
* Routes work to the correct decision surface.
* Reduces rework across drafting, review, and maintenance cycles.

### V10 Platform Innovations

#### Stonewall Brain v10.4

The core intelligence layer reached v10.4 with:

* **Deterministic mode routing** — Analytical narration and operational execution strictly separated at the architecture level
* **Retrieval cascade architecture** — Fixed resolution path: skill → case codex → artifact codex → cast codex → phenomenology registry → corpus compendium → corpus search
* **Codex + Registry stack** — Identity, timeline, case-state, and pattern indexes as a single retrieval stack
* **Formation patterns** — Meta-patterns that instantiate retroactively across the entire corpus
* **Full book-mode chapter generation** — 700+ word sections, flowing prose for long-form output
* **Tactical mode** — Skill dispatch routing table for operational execution

#### Corpus Hardening

The document corpus underwent a major hardening sweep, achieving:

* 1,190 manifest rows, 1,106 active, 724 analyzed (65.5% coverage)
* **0 validation errors** on strict mode after systematic repair
* PDF sidecar repair, OCR fallback, and verbatim normalization applied across the corpus
* Full deduplication and canonical precedence enforcement

#### Stonewall CLI

A full command-line interface for corpus and case management:

```bash
python stonewall.py stats          # Corpus health and coverage summary
python stonewall.py find <query>   # Full-text search with result limiting
python stonewall.py case <id>      # Case posture and timeline
python stonewall.py pattern <id>   # Pattern detail and cross-references
python stonewall.py validate       # Ontology validation against schema
python stonewall.py doctor         # Corpus health diagnostics
```

Supports JSON output (`--json`) and ontology validation against a machine-readable schema.

#### Multi-Platform Architecture

The platform now spans five synchronized surfaces:

| Surface | Role |
|---|---|
| **GitHub** | Canonical archive and version control |
| **Notion** | Live case posture, tasks, email triage, command center |
| **OneDrive Personal** | Staging and brain deployment |
| **OneDrive Firm** | Read-only evidence reservoir |
| **Local synthesis layer** | Fast corpus recall and PowerShell retrieval |

#### Document Intelligence Pipeline

Full automation stack supporting the complete document lifecycle:

```
Source → Ingestion → Processing → Notion Sync → AI Tagging → QC & Reporting
```

Supported formats: PDF, DOCX, XLSX, CSV, TXT, HTML, EML, MSG

Features AI-powered semantic email classification, property auto-fill, and automated QC sweeps with cross-system consistency validation.

#### PR Verification System

Automated pre-PR gates with:

* `verify_all.py` as the canonical verification gate
* `pr_checklist.py` generating structured summary blocks
* CI/CD integration on every push and pull request
