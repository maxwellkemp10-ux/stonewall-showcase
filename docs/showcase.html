Stonewall turns legal document chaos into operator-grade leverage.

Stonewall is a legal document intelligence control plane built by a solo litigation attorney. It turns flat files, sidecars, retrieval discipline, and static publishing into one legible operating system for artifacts, deadlines, workflow readiness, and daily portfolio control, without exposing client names, live captions, or privileged content.

Commercial Logic

Why the product story lands

Stonewall stops sounding like “AI for legal” the moment the workflow layer becomes visible. The archive is useful because it can surface today’s runway, stage report-ready packets, tighten witness preparation, and preserve institutional memory without asking the operator to rebuild context manually every morning.

The buyer-legible outcomes are simple: daily dossier, deadline intelligence, DataGavel packet readiness, Notion command visibility, and live deposition tailoring from the same indexed corpus.
🗃️

Notion as operator layer

The repository remains the durable archive; Notion becomes the live control board where deadlines, archive links, matter posture, and task runway become legible in seconds. That separation is what makes the system feel operational instead of theoretical. 

repo = durable truth
notion = live command surface
public docs = external proof surface
📐

DataGavel workflow readiness

Stonewall is designed to make structured report workflows easier to feed. Chronology, treatment trail, and damages notes can already be staged before the specialist report workflow begins, which turns packet assembly into a repeatable operating motion. 

records pulled
-> chronology checked
-> treatment ledger aligned
-> damages notes staged
-> report packet ready
🎯

Live deposition tailoring

Prior statements, chronology gaps, and issue clusters are retrievable quickly enough to tighten the next section of questioning while the witness is still in the room. That is a different class of usefulness than retrospective storage. 

python scripts/stonewall.py find "corporate representative"
python scripts/stonewall.py timeline --start 2025-02-01 --end 2025-02-28
python scripts/stonewall.py show A1104
🧠

AI recall with governance

The Stonewall brain is a versioned retrieval discipline, not a black box. Codex files route the assistant to the right source stratum, then force fresh reading before confident claims, which keeps the intelligence layer aligned with the corpus instead of drifting away from it. 

brain/
  cast_codex.md
  cases_codex.md
  corpus_mapping.md
  legal_standards.md

System Shape

One archive, multiple working surfaces

The architecture is persuasive because the same source of truth drives the CLI, the operator view, the public showcase, and the documentation layer. The platform does not need a private application server to make its reasoning visible.

The public build strips out names and privileged text, but it leaves the shape of the operating system intact so an outside reader can still see how it works.
