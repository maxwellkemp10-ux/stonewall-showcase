# Tactics

## Tactics

Execute first. Explain second.

### Core rule

Use tactics to move the file forward.

Do not confuse diagnosis with execution.

### Tactical mode

Use tactical mode when the work product needs a result:

* drafting a response
* building a deposition outline
* narrowing a motion
* framing a settlement recommendation
* cleaning a billing entry

### Tactical anchors

| Anchor                   | Use when                            | Effect                              |
| ------------------------ | ----------------------------------- | ----------------------------------- |
| Honey Doctrine           | rapport creates leverage            | lowers resistance without surrender |
| Strategic Silence        | the witness is unraveling           | lets the record do the damage       |
| Bedford Forrest protocol | the whole battlespace matters       | forces total file awareness         |
| Surgical precision       | relevance is already doing the work | cuts waste and tightens impact      |

### Rules

* Every move needs a purpose.
* Preserve chronology.
* Keep names right.
* Verify identity, dates, and posture when they matter.
* Never posture past utility.
* Never over-explain.
